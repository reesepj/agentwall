# Agentwall brand starter

This repo now includes a practical, editable brand starter for Agentwall.

## Naming

- Product name: **Agentwall**
- Use exactly this casing in docs/UI/CLI output.
- Avoid variants like `AgentWall`, `agent wall`, or `AGENTWALL` except for all-caps headings.

## Logo assets

Located in `public/assets/brand/`:

- `agentwall-logo-primary.svg` — default lockup with wordmark + tagline
- `agentwall-logo-mark.svg` — square app/icon mark
- `agentwall-logo-monochrome.svg` — single-color lockup for constrained surfaces
- `favicon.svg` — browser/favicon mark for public surfaces
- `icon_idle.svg` — low-activity / standby runtime state
- `icon_flow.svg` — active flow / simulation state
- `icon_active.svg` — healthy live runtime state

## Color direction

- **Midnight** `#0E152A` (primary dark)
- **Blue** `#5B8CFF` (trust / intelligence accent)
- **Teal** `#00C2A8` (secure/action accent)
- **Slate** `#4B5B78` (secondary text)

## Font direction

Recommended stack:

1. **Inter** (primary UI and product wordmark)
2. `Segoe UI`
3. `system-ui`
4. `sans-serif`

Usage guidance:

- Headlines and product naming: Inter 600–700
- Body/UI text: Inter 400–500
- CLI/terminal docs: a monospace stack (`ui-monospace`, `SFMono-Regular`, `Menlo`, `monospace`)

## Tone direction

- Technical and confident
- Security-first without fear-marketing
- Operationally clear (show exact commands and outcomes)
