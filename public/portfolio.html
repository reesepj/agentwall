Agentwall: The Runtime Control Plane for AI Agents

Agentwall is a provenance-aware enforcement layer between autonomous agents and real-world action surfaces. It evaluates actions before execution, enforces policy across egress/tools/browser/content/identity/governance, and emits operator-grade decision evidence.

Execution-time controls Default-deny egress Human-in-the-loop approvals Forensic auditability

Problem

Prompt safety doesn’t control runtime behavior. Once agents can fetch, click, run, and send, you need an enforcement plane that understands risk and provenance.

Thesis

Put policy at the exact boundary where intent becomes action. Agentwall decides, logs, and escalates where needed.

Product Surfaces

  • Security Console (operator UI)
  • Policy + inspection APIs
  • Approval queue + session controls
  • Evidence ledger + task graph

Key Capabilities

  • Provenance + trust labels
  • SSRF/private-range egress blocking
  • DLP + redaction controls
  • Actor-scoped policy rules

CLI Story

  • npm run onboard config bootstrap
  • npm start runtime
  • npm test verification
  • Policy hot-reload from disk

Rollout Model

  • Observe: mirror risky calls, fail-open, collect evidence.
  • Enforce Critical: block metadata/private-range egress and other high-confidence threats.
  • Expand: enforce tool/browser/identity controls with approval routing.
  • Harden: add watchdog containment, manifest re-approval, SOC pipeline integration.

Why It Matters

Agentwall gives teams the middle path between unsafe autonomy and unusable lockdown. It preserves shipping speed while adding enforceable guardrails, explicit approvals, and audit evidence good enough for incident response, compliance, and operator trust.
